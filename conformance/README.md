# Conformance corpus

Language-neutral golden fixtures that **every** IRG implementation (JS, Python,
Rust, LangChain, …) must reproduce identically. They pin down the deterministic,
I/O-free parts of the contract so polyglot ports can be proven equivalent rather
than merely "looks similar."

These fixtures are not owned by any one implementation — each port runs them
against its own code and diffs exactly.

## Fixtures

### `fixtures/citation-apply/*.json`
Inputs and expected outputs for `citationApply` (Citation_Application.md §6/§11):

```jsonc
{
  "name": "...",
  "draft_with_tags": "...prose with <citation ref=\"cit_N\">…</citation>...",
  "citable_set": [ { "handle": "cit_N", "uuid": "...", ... } ],
  "expected": { "validated_prose": "...", "references": [ ... ], "stats": { ... } }
}
```
`citable_set` carries explicit `uuid`s so this corpus tests the validation
contract independently of UUID derivation.

### `fixtures/citation-id/cases.json`
`claim_key → derived UUID v5` pairs (generated by the JS reference). A port's
`deriveClaimUuid` must reproduce every one, using the fixed namespace recorded
in the file.

## Who runs them

| Implementation | Runner |
|---|---|
| JS (`api-impl-js`) | `test/test-citation-conformance.js` |
| Python (`api-impl-py`) | `tests/test_citation_conformance.py`, `tests/test_citation_id.py` |
| Rust (`api-impl-rs`) | `tests/conformance.rs` (`cargo test`) |
