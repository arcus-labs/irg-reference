# IRG — Python implementation (`api-impl-py`)

A Python port of the **IRG reference implementation**. The canonical, full
implementation lives in [`../api-impl-js`](../api-impl-js); this directory is
part of the polyglot "Rosetta-Stone" goal — *here's how to implement IRG in
your stack* — and is validated against the shared, language-neutral
conformance corpus in [`../conformance`](../conformance).

## Scope (this milestone)

The **pure citation conformance core** — the deterministic, I/O-free logic that
every port MUST reproduce identically:

| Module | Mirrors (JS) | Purpose |
|---|---|---|
| `irg/citations/citation_id.py` | `core/citations/citation-id.js` | deterministic claim UUID (RFC-4122 v5 over `claim_key`) |
| `irg/citations/citation_format.py` | `core/citations/citation-format.js` | `<citation>` tag parse / sanitize / serialize |
| `irg/citations/apply.py` | `core/citations/apply.js` | validate → resolve → renumber → build `references[]` |
| `irg/citations/build_citable_set.py` | `core/citations/build-citable-set.js` | assemble the citable set from verify/recall results |
| `irg/citations/quality_metrics.py` | `core/citations/quality-metrics.js` | ALCE-style citation recall / precision / f1 |
| `irg/external_fact_check/claimreview.py` | `core/external-fact-check/claimreview.js` | schema.org `ClaimReview` interop projection |

**Not yet ported** (future milestones): the reasoning-graph executor and nodes,
the multi-provider LLM layer, and the fact-store (DuckDB / fetch / verify).
Those are the non-deterministic, I/O-bound parts and are *not* part of the
conformance contract.

## Conformance

`tests/test_citation_conformance.py` runs the **same** golden fixtures the JS
implementation runs (`../conformance/fixtures/citation-apply/`) and asserts the
Python `apply_citations` output is byte-comparable to the expected result.
`tests/test_citation_id.py` checks `derive_claim_uuid` against the shared
`../conformance/fixtures/citation-id/cases.json` (UUIDs generated by the JS
reference). If these pass, the port is provably equivalent on the core.

## Running the tests

Stdlib only — no dependencies, no virtualenv required (Python 3.8+).

```bash
cd api-impl-py
python3 -m unittest discover -s tests -p "test_*.py"
# or
python3 tests/run_all.py
```
